## プレビューの追加方法

技術データに視覚的なプレビューを追加することで、技術ドキュメントの理解度が大幅に向上します。

### プレビュー追加の基本ルール

1. **対象**: UI/デザイン系の技術（レイアウト、デザイン、UIコンポーネントなど）
2. **形式**: HTMLとTailwind CSSのみ使用
3. **サイズ**: コンパクトで見やすいサイズ（h-32, h-48など）
4. **省略可**: アニメーション系など静的プレビューで表現困難な技術

### プレビューの追加例

```typescript
{
  slug: 'gradient-button',
  title: 'グラデーションボタン',
  // ... 他のフィールド ...
  preview: `<div class="flex justify-center">
    <button class="bg-gradient-to-r from-blue-600 to-purple-600 text-white px-8 py-3 rounded-lg hover:shadow-lg transition-all">
      クリックボタン
    </button>
  </div>`,
  relatedTech: []
}
```

### プレビュー作成のコツ

- **色**: 実際のデザインで使用されている色を使う
- **レイアウト**: グリッド、フレックスボックスなど実装と同じ構造
- **サイズ**: プレビューエリアに収まるように調整
- **アイコン**: SVGを直接埋め込む（簡潔に）
- **テキスト**: ダミーテキストは短く簡潔に
